🏆 랭킹 시스템 도입 검토 Cowboy Duel · Firebase 무료(Spark) 티어 기준

✅ 결론: 무료티어로 충분히 가능합니다. 추가 비용 사실상 0원. 이미 쓰는 Firebase Realtime DB + 익명 로그인만 켜면 "온라인 승수 글로벌 랭킹"을 만들 수 있어요. (반나절~하루 작업 규모)

1. 지금 상태

  • 백엔드: Firebase Realtime Database (cowboy-duel-doonghwi) — 온라인 방 대전 이미 작동.
  • 플레이어 식별: 지금은 세션마다 랜덤 ID(영속 X). 랭킹은 누적이라 기기/계정 단위로 유지되는 ID가 필요.
  • 닉네임: 입장 시 입력(빈칸이면 랜덤). 랭킹 표시명으로 재사용 가능.

2. 랭킹에 필요한 4가지

요소방법무료?
영속 식별Firebase 익명 인증(Anonymous Auth) → 기기별 고정 uid무료
누적 지표승수(wins) / 점수 — 온라인 승리 시 +1 (트랜잭션)무료
정렬 조회RTDB orderByChild('wins').limitToLast(50) → 상위 50명무료
부정 방지보안 규칙(본인 것만 쓰기) — 완벽친 아님(아래 참고)부분

3. 데이터 모델 (제안)

/leaderboard/{uid} = {
  "name": "방랑객42",
  "wins": 17,
  "updatedAt": 1780000000000
}
  • 온라인 듀얼 종료 시 승자 uid의 wins를 트랜잭션으로 +1.
  • 랭킹 화면: orderByChild('wins')로 정렬해 상위 N + "내 순위" 강조.
  • vs 컴퓨터 승리는 검증 불가 → 랭킹엔 온라인 승수만 반영(추천).

4. 보안 규칙 (테스트모드 → 정식)

{
  "rules": {
    "leaderboard": {
      ".read": true,
      "$uid": {
        // 본인 것만, 로그인 상태에서만 쓰기
        ".write": "auth != null && auth.uid === $uid",
        "wins": { ".validate": "newData.isNumber()" }
      }
    },
    "rooms": { ".read": true, ".write": true }
  }
}

현재 RTDB는 테스트 모드(30일 오픈)라 어차피 만료 전에 정식 규칙이 필요해요. 랭킹 도입은 그 작업과 묶으면 효율적입니다.

5. 무료(Spark) 한도 — 이 게임엔 여유

항목무료 한도랭킹 사용량
RTDB 동시접속100여유 캐주얼 규모
RTDB 저장/전송1GB / 월 10GB여유 랭킹 데이터 KB 단위
익명 인증무제한 무료OK
Cloud Functions(서버검증)이제 Blaze 필요기본 랭킹엔 불필요

6. 부정행위(치팅) 현실

  • 클라이언트가 wins를 직접 올리는 구조라, 마음먹으면 자기 점수 조작 가능(서버 검증 없음).
  • 완화: 보안 규칙으로 "본인 uid만 / 숫자만 / 한 번에 +1만" 정도 제한 가능 → 캐주얼엔 충분.
  • 완전 차단은 Cloud Functions로 서버측 검증(경기 결과를 서버가 판정) 필요 → Blaze 플랜. 인기 생기면 그때 도입.

7. 적용 단계 (제안)

1
Firebase 콘솔에서 익명 인증 켜기(클릭 1회) + 앱에서 signInAnonymously()
2
온라인 승리 시 /leaderboard/{uid}의 wins +1, 닉네임 저장
3
홈에 "랭킹" 버튼 + 랭킹 화면(상위 50, 내 순위 하이라이트)
4
정식 보안 규칙 배포(테스트모드 만료 전 필수 작업과 함께)
5
(확장) 일간/주간 랭킹은 키에 날짜 추가(/leaderboard/2026-06/{uid})
